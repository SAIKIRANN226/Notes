-----------------------------------------------------------

CI (Continous integration)
-----------------------------
What are we doing in CI ? We are integrating the code like we have developers and they are continously pushing the code to the git, as a devops engineer we need to take that code and check wether the code is compiling properly (or) not and building the code generally we get errors while building the code, for that scanning and unit testing and saving at one place, this is called continous integration process, If this process succeded then only we go for deployment which is "CD"

1. First catalogue should be in git
2. Write Jenkinsfile for this, what should be stages in jenkinsfiles ?
   a) clone the code
   b) unit testing
   c) scans like (sonar scan)
   d) installing dependencies
   e) build (zip file)
   f) push the zip file to the "nexus" which is repository where we keep our "artifacts"

CD (Continous deployment)
----------------------------
1. deploy to dev
2. testers will test the functional cases
3. finally publish the results
