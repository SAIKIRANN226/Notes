Terraform
-----------
Is a popular IAC(infrastructure as a code)

We configured our project manually,shellscript,ansible.but we prefer ansible as a configuration management if it is large project.

Why we dint prefer manuall configuration?
------------------------------------------
1.Because it is time taking process,because manully logging in the servers is timetaking
2.We get more human errors
3.Ansible is it can connect to more number of servers easily

Version control
----------------
If we keep code in the GIT,we can maintain history,we can also get back to the previous version incase of any production failure

Consistent infrastructure
--------------------------
Same code will work with all environments like dev,sit,uat,prod.real time scenario like code is wokring in dev but not in prod,for this issue we use terraform

Automated infrastructure(CRUD)
-------------------------------
Create the infra
Read the infra
Update the infra
Delete the infra

Inventory management
---------------------
For example if the project is big.when you login to aws we can see more services like multiple VPC's,multiple route 53,multiple EC2,multiple load balancers,CDN's,s3 buckets etc.if the client ask for the full details of the project we cannot show one by one in the aws and  giving the report is very tough and that is also not a good practice,instead of showing in the aws console we can show in VS it is like folder structure everything is in neat format.

Cost optimization
------------------
1.Create when required,stop when not required
2.Automatic dependency management,for example if we need to create route53,ec2 servers should
  be in running take those ip and then paste in the route53,that means route53 is depending 
  on ec2 servers,that means we can create resources and terraform will takecare of 
  dependencies.
3.Terraform modules--->you can create terraform code as modules,other project can also use it
  without writing from the scratch

4.Terraform is also a hybrid cloud that means you can create insfra in aws,azure,GCP not only 
  cloud terraform can work with any other popular tool in the market,you can see in the 
  terraform providers,it can work with everything

The above all are the terraform understanding(interview question).it is a declarative way of creating infrastructure,what is declarative--->whatever you write you get it if you write correct syntax,easy syntax and no need to follow sequence

If anybody edit manually in the aws console,and we find it is very difficult to trace who did it,but in terraform if anybody did changes and we can restore infrastructure it by just one command,then we can discuss who did it later

Terraform commands
-------------------

1.terraform init --> initialize the provider,it will download the aws plugins or provider
  ".terraform folder" is created to download the provider

2.terraform plan --> terraform will calculate full details and show it or it is just a plan
  it will not create

3.terraform apply --> now it will create

4.terraform apply -auto-approve --> not ask for prompt

5.terraform destroy -auto-approve --> not ask for prompt

6.All the above commands should be run in the gitbash
