What is the difference between free-style job and pipeline job?
-------------------------------------------------------------------
1. Free-style
--------------
Free-style job is no one is preferring now, it is very easy to do everything from the console, anybody can do the changes and we dont know who did what? and it is very difficult to rollback to the original version, thats why everybody is moved to pipeline, So now we should understand jenkins pipeline just search "jenkins pipelines" click on the first link and know about the pipeline syntax

2. Pipelines
-------------
Now create a new job with "pipeline" and use the default script of hello-world in the "pipeline script" and you can add any number of stages, But we prefer "pipeline script from SCM" because anybody can change the code from the pipeline script, Our source code(SCM) is git if we put this pipeline in git then jenkins will take that code from the git and run it this is called as "gitOps" Note:- Put the https URL of jenkins file from the git in the jenkins server

Raw syntax of the pipeline(interview question)
-------------------------------------------------
pipeline {
    agent any
	
	stages {
	   stage('Build') {
	      steps {
		    echo 'Building..'
		  }
	   }
	   stage('Test') {
	      steps {
		    echo 'Testing..'
		  }
	   }
	   stage('Deploy') {
	      steps {
		     echo 'Deploying..'
		  }
	   }
	}
}

What is agent?
----------------
Take the example of 1 person ---> 1 acre of land ; 100 acres ---> lot of resources
owner ---> emploee resources ---> distribute the work

If you have single project you can use single sever(jenkins-master), but if you have multiple projects one single jenkins server cannot handle the load so we have AGENT(which act like a slave) "MASTER-AGENT". So
create one more instance for agent and install java same command which is used in the jenkins server, Now configure this agent in jenkins server by going to the manage jenkins option/nodes/new-node(keep name as per your wish)/generally jenkins configuration will be saved in "cd /var/lib/jenkins/" so in the sameway you need to create a directory while creating agent "/var/jenkins-agent". Number of executers here we put 3 as per the instance.

1. Master asking agent to work ---> "launch agents via ssh" master will connect to agent through ssh
2. Employee coming to master and asking for work
